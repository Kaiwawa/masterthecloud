GCP Resource Hierarchy

How Google Cloud organizes resources and how policies are inherited top-down.

🏢

Organization &
Folders

The root node of the GCP hierarchy. Policies defined here automatically cascade down to every folder, project, and resource beneath them.

Organization Node

The absolute top level. Represents your company (e.g., example.com). Tied to a Google Workspace or Cloud Identity domain. Centralizes visibility and auditing across all projects.

Folders

Used to group projects. You can map them to your business structure (e.g., "Engineering", "Marketing", or "Production", "Development"). Folders can contain other folders or projects.

📁

Projects &
Resources

The functional levels where work actually gets done. Billing, APIs, and specific resource limits are tracked at the project level.

Projects

The fundamental trust domain. ALL resources must belong to exactly one project. Projects are the basis for enabling APIs, setting up billing, and managing specific permissions.

Resources

The lowest level (e.g., a Compute Engine VM, a Cloud Storage bucket, a BigQuery dataset). They inherit policies from the Project, which inherits from the Folder, which inherits from the Organization.

Shared Responsibility Model

Google Cloud Platform: Shared Security.

👤

Customer
Responsibility IN the Cloud

Content & Data

The customer is responsible for data classification, encryption at the application level, and access control (Cloud IAM).

Service Configuration

Configuring firewall rules, assigning roles, and managing application-level security.

☁️

Google Cloud
Responsibility OF the Cloud

Hardware & Networking

Google manages physical security, hardware lifecycle, and the global high-speed network.

Storage & Encryption

Google provides default encryption at rest and in transit, and manages the underlying storage stack.

Compute & Services

Virtual machines, serverless, and orchestration.

💻

VMs &
Servers

  • ✦ Need full OS control? Use Compute Engine (IaaS) for total flexibility.
  • ✦ Need orchestration at scale? Use GKE or Autopilot for complex microservices.
⚡

Serverless &
Apps

  • ✦ Need raw code or containers? Use App Engine (PaaS) or Cloud Run (Serverless containers).
  • ✦ Event-driven snippets? Use Cloud Functions for lightweight execution.

💻 Compute Options Cheat Sheet
